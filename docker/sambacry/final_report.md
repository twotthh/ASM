# 📄 SambaCry (CVE-2017-7494) 취약점 탐지 및 조치 보고서

## 1. 취약점 개요 (Description)
* **CVE 번호:** CVE-2017-7494 (SambaCry)
* **개요:** Samba 시스템에서 인증되지 않은 사용자가 원격에서 악의적인 공유 라이브러리를 로드하여 코드를 실행할 수 있는 원격 코드 실행(RCE) 취약점입니다.

## 2. 1단계: Docker 기반 취약 서버 구축
* **사용 이미지:** `vulnerables/cve-2017-7494`
* **구동 포트:** `445/tcp`
* 로컬 Docker 컨테이너 환경에서 SambaCry 취약 서버를 정상적으로 구동시켰습니다.

## 3. 2단계: 취약점 탐지 결과
* **Nmap 스캔 명령어:** `sudo nmap -sV -O -p 445 127.0.0.1`
  * 스캔을 통해 445번 포트 오픈 및 `Samba smbd 3.X - 4.X` 버전, 리눅스 OS 환경을 식별했습니다.
* **Nuclei 스캔 명령어:** `nuclei -target 127.0.0.1:445 -id cve-2017-7494`
  * Nuclei 스캐너를 구동하여 `[cve-2017-7494]` 취약점이 호스트에 존재함을 성공적으로 탐지하고 마킹했습니다. (구동 과정은 단톡방에 올린 영상으로 증명 완료)

## 4. 3단계: 관리자 대응 조치 가이드 (Mitigation)
1. **최신 패치 적용:** Samba 버전을 보안 패치가 완료된 최신 버전(4.6.4, 4.5.10, 4.4.14 이상)으로 즉각 업데이트할 것을 권고합니다.
2. **임시 완화 조치:** 즉각적인 패치가 불가능한 경우, 삼바 설정 파일(`/etc/samba/smb.conf`)의 `[global]` 섹션에 아래 설정을 추가하여 파이프 지원을 비활성화한 후 서비스를 재시작합니다.
```text
   nt pipe support = no
